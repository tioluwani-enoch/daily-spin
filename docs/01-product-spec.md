# 01 · Product Spec

This document is the canonical description of what Daily Spin does. Every feature here maps to one module under `docs/08-modules/`. If a feature is not in this document, it does not belong in the product.

---

## The daily ritual

The product is shaped around two distinct moments in a user's day.

### The morning open (60–90 seconds)

The user opens Daily Spin once shortly after waking. The screen they see, in order from top to bottom:

1. **Today's pick.** One track from their own library, with a one-sentence reason. Tap to play (opens Spotify). Dismiss to skip and reveal another. The reason is generated by Spin Companion using the user's listening data.
2. **New from your artists.** Up to five releases from the user's watchlist artists, dropped in the last 14 days. Each shows artist, release title, type (single, EP, album), and a tiny audio fingerprint badge. Tap to play. Dismissable.
3. **Capture inbox.** Items the user added since the last open. Each item shows the song, where it came from, and three actions: add to playlist, save for later, dismiss.

That is the entire morning ritual. No scroll past these three things.

### The session open (variable)

Later in the day, the user opens the app to pick music for an activity (working, gym, hosting friends) or to maintain a playlist. The same home screen is up top, but now they go deeper:

- **Playlist Health view.** All their playlists ranked by attention needed (drifting, stale, dying, healthy). Tap any playlist to enter the curation view.
- **Spin Companion chat.** Always available as a sheet from any screen. The user can ask questions like "what should I play for a long drive" or "remove the songs that don't fit the vibe of this playlist".

## The five core features

### 1. Morning Pick

A single track surfaced each day from the user's saved library, picked to encourage rediscovery rather than novelty.

**Inputs:** saved library, listening history (last 90 days), audio features per track, saved date per track.

**Output:** one track ID, one short reason string (generated by Spin Companion), a "why this" expandable detail.

**Algorithm summary:** weighted random selection biased toward tracks saved more than 30 days ago, played fewer than three times in the last 90 days, and similar in audio features to what the user has been into the last seven days. Full algorithm in `docs/08-modules/morning-pick.md`.

**Edge cases:** a fresh user with no listening history falls back to "least-played saved tracks." A user who dismisses the pick gets up to two more attempts before the day ends.

### 2. Artist Watchlist

A user-curated list of 30–100 artists. Daily Spin polls Spotify's release feed daily and surfaces only drops from this list. No global new-music feed, no algorithmic suggestions.

**Inputs:** user's watchlist (manual), Spotify's `artist albums` endpoint per artist, release dates.

**Output:** a feed of new releases, each tagged by release type, with a freshness window (default 14 days).

**Edge cases:** an artist who drops six singles in a week shows the most recent two with a "+4 more" link. Compilations and re-releases are filtered unless the user opts in per artist.

### 3. Capture Inbox

A frictionless one-tap capture for songs the user discovers anywhere. Multiple input paths:

- A web bookmarklet that grabs the page URL plus any embedded Spotify metadata.
- A share-sheet target on iOS and Android (Phase 2, requires PWA install).
- A paste field inside the app for raw URLs or "Artist – Track" strings.
- A Spin Companion command ("save Caroline Polachek's Bunny Is a Rider for me").

**Output:** an entry in the inbox with source, captured-at timestamp, and a parsed track ID once we resolve it.

**Triage actions:** add to playlist (with playlist-health-aware suggestions), save to library, save for later (a "deep crate" the Morning Pick can draw from), dismiss.

**Edge cases:** unresolvable captures (e.g., a Bandcamp track not on Spotify) are kept as text-only and surfaced for manual handling.

### 4. Playlist Health

The product's secret weapon. Each playlist is monitored on three dimensions:

- **Drift.** How far the playlist's recent additions deviate from its original sonic fingerprint. Computed as cosine distance between the audio-feature centroid of the first N songs and the centroid of the last M songs.
- **Staleness.** Time since last edit weighed against play frequency. A playlist played daily but unedited for six months is healthy. One unedited and unplayed for two months is stale.
- **Death.** Playlists with zero plays in 90+ days surface as candidates for archive or deletion.

**Output per playlist:** a health label (healthy, drifting, stale, dying), suggested actions, and — for drifting playlists — concrete add/remove suggestions.

**Curation view:** for any playlist, the app proposes 3–5 additions (drawn from the user's library or related artists in their watchlist) that match the original fingerprint, plus 1–3 outlier songs flagged for removal. Every suggestion shows its computed similarity score.

### 5. Weekly Recap

A Sunday-morning journal entry, not a leaderboard. Generated by Spin Companion using the week's listening data.

**Contents (typical):**

- Top tracks by play count, with a one-line note on each
- Mood arc through the week (e.g., "ambient on Monday and Tuesday, harder edits Wednesday onward")
- Returns: songs you came back to after a long absence
- Drifts: playlists that moved away from their fingerprint this week
- One open question for the user (e.g., "your jazz listens dropped to zero this week — intentional or just busy?")

**Format:** a single scrollable page, not a deck of cards. Saveable as a static snapshot the user can revisit.

## Spin Companion (cross-feature)

Spin Companion is not a separate feature so much as the connective tissue between all the others. It is available as a chat sheet from any screen, and it also writes the short reason strings, recap copy, and intro text that appears throughout the UI.

**Things the user can ask:**

- "Why did you pick this song today?"
- "Give me something for a long walk in the rain."
- "What's drifting in my playlists right now?"
- "Add this to my Sunday Mornings playlist if it fits, otherwise tell me why it doesn't."
- "What did I play three weeks ago that I haven't returned to?"
- "Make me a 30-minute playlist that bridges from ambient to UK garage."

The Companion answers using the user's actual data via tool calls. It never invents facts about songs or artists; if it does not know, it says so. Full design in `docs/06-spin-companion.md`.

## User flows

The four flows that matter most. Each one is a target for usability work.

### Flow A: First-time setup

1. User lands on `/`, sees a one-screen pitch and a "Sign in with Spotify" button.
2. OAuth handshake.
3. Initial library backfill kicks off in the background; user sees a progress card.
4. User is dropped into a one-screen "Pick your watchlist" flow. The app suggests 30 artists drawn from their top-played artists; user can add, remove, or skip and curate later.
5. First Morning Pick appears.

### Flow B: Daily morning open

Already described above. Single screen, three sections, average dwell time 60–90 seconds.

### Flow C: Capture and triage

1. User hears a song somewhere; hits share or paste; the song lands in the inbox.
2. Next morning open, they see it in the inbox section.
3. One tap on "add to playlist" opens a sheet with playlist suggestions ranked by sonic fit. User picks one, song is added via the Spotify API, sheet dismisses.

### Flow D: Playlist cleanup

1. User opens the Playlist Health view.
2. They tap a playlist labeled "drifting".
3. They see the curation view: 3 suggested adds, 2 suggested removes, the playlist's fingerprint visualized as a small radar chart.
4. They accept some, reject others. Changes write back to Spotify.

## Out of scope (v1)

- Social features
- Audio playback inside Daily Spin
- Apple Music integration (Phase 3 at earliest)
- Local file libraries
- Lyric search / display
- Concert ticket integration
- AI-generated music or remixes
