# 11 · Glossary

Terms that show up across the codebase and the docs. When in doubt, this file is canonical.

---

**Ambient theme**
The interface's color palette and motion timing, derived from whatever track is currently playing. Driven by CSS variables on `:root`. See `docs/07-design-system.md`.

**Audio features**
The numeric vector Spotify provides per track: energy, valence, tempo, danceability, acousticness, instrumentalness, liveness, speechiness. Daily Spin uses 7 of these (excludes speechiness) and normalizes tempo to [0, 1].

**Backfill**
The initial bulk fetch of a user's library, playlists, and listening history after they sign in for the first time. Runs once per user as a background job; subsequent updates are diffs.

**Capture**
A song the user saved into Daily Spin's inbox via bookmarklet, share sheet, paste field, or Companion command. Captures start in `pending` resolution state and are resolved to a Spotify track id when possible.

**Centroid**
The mean of a set of audio-feature vectors. Used to define playlist fingerprints (a core centroid and a recent centroid per playlist).

**Companion / Spin Companion**
The conversational layer powered by Claude. Available across the app as a chat sheet; also generates the prose for Morning Pick reasons, new release blurbs, and weekly recaps. See `docs/06-spin-companion.md`.

**Core (of a playlist)**
The first N tracks of a playlist (default N=30). Defines what the playlist *is*, sonically. The core centroid is computed from these.

**Cosine distance**
`1 - cosineSimilarity`. Used everywhere we compare two audio-feature vectors. Range: [0, 2], but in practice we work in [0, 1] because all our vectors are non-negative.

**Curation view**
The per-playlist editing screen that shows fingerprint visualization, suggested adds, and suggested removes. Owned by the Playlist Health module.

**Daily ritual / morning ritual**
The first screen the user sees when they open the app each morning. Three sections: today's pick, new releases, capture inbox. Designed to take 60–90 seconds.

**Deep crate / for-later**
A capture triage outcome that moves a song into a "save for later" bucket. The Morning Pick algorithm gives a small boost to deep-crate tracks that have been there >14 days.

**Drift / drift score**
The cosine distance between a playlist's core centroid and its recent centroid. Higher means the playlist has moved away from its original sonic identity. Threshold for the "drifting" health label is 0.15.

**Dying (playlist)**
A playlist with no plays in 90+ days and no edits in 90+ days. Surfaced for archive consideration; never deleted automatically.

**Fingerprint**
A playlist's sonic identity, expressed as two centroids (core and recent) plus a drift score. Stored in `playlist_fingerprints`. Recomputed when the playlist changes.

**Freshness window**
The time horizon within which a release is considered "new" for the morning ritual feed. Default: 14 days from `release_date`.

**Health label**
The categorical status of a playlist: `healthy`, `drifting`, `stale`, `dying`. Computed by a small decision tree using drift score plus play and edit recency.

**Listening history**
The append-only log of plays from `/me/player/recently-played`. Spotify only exposes the last 50 plays per user, so we sync every 6 hours and accumulate locally.

**Module**
A self-contained feature folder under `src/modules/`. Each module has its own data access, business logic, UI, and a thin public API in `index.ts`. Modules do not import from each other. See `docs/02-architecture.md`.

**Morning Pick**
The single track surfaced each morning from the user's saved library. Picked by a scoring algorithm that weights save recency, underplay, affinity to recent listening, and novelty. See `docs/08-modules/morning-pick.md`.

**New release**
An entry in the morning ritual's new releases section. Comes from a watchlist artist and falls within the freshness window.

**Pick reason**
A one-sentence string explaining why a Morning Pick surfaced today. Generated by Spin Companion (with a deterministic template fallback) and cached in `daily_picks.reason`.

**Recap (weekly)**
A Sunday-morning structured-data-plus-prose snapshot of the user's listening week. Stored in `weekly_recaps`. See `docs/08-modules/weekly-recap.md`.

**Recent (of a playlist)**
The last M tracks of a playlist (default M=10). Used to compute the recent centroid and detect drift.

**Resolution**
The process of turning a raw capture input (URL or text) into a Spotify track id. Happens asynchronously after capture creation. May fail; failed resolutions become `unresolvable` and require manual user input.

**Sonic fingerprint**
See *Fingerprint*.

**Spin Companion**
See *Companion*.

**Stale (playlist)**
A playlist not edited in 60+ days and not played in 30+ days. Surfaced for cleanup.

**Taste graph**
The user's set of musical signals: their saved library, their listening history, and their watchlist artists' catalogs. Spin Companion never recommends music outside this graph.

**Triage**
The act of processing a capture from the inbox. Four actions: add to playlist, save to library, for-later, dismiss.

**Underplay**
A scoring axis for the Morning Pick. A track is underplayed if it has been played 0–2 times in the last 90 days. Higher underplay → stronger candidate for surfacing.

**Watchlist**
The user's curated list of artists Daily Spin polls daily for new releases. Manually maintained. Capped (soft) at 100 artists per user. See `docs/08-modules/artist-watchlist.md`.
